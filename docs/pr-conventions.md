# PR Conventions

이 Repository의 Pull Request 관습을 정의합니다.

## 이 Repository의 특수성

이 Repository는 JK의 개인 Repository로, 다른 Repository와 다른 워크플로우를 따릅니다.

| 항목 | 이 Repository | 다른 Repository |
|------|---------------|-----------------|
| PR 작성자 | AI Agent | 엔지니어 |
| 리뷰어 | JK 1인 | 팀 엔지니어들 |
| 승인 방식 | JK 단독 승인 | 팀 리뷰 후 승인 |

## PR 작성 규칙

### 작성자 표시

PR은 AI Agent가 작성하며, 커밋 메시지에 Agent 이름을 명시합니다:

```
Co-Authored-By: Atlas <atlas@jk.agent>
```

### PR 제목 형식

```
<type>: <description>
```

Types:
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `refactor`: 리팩토링
- `chore`: 기타 작업

### PR 본문 형식

```markdown
## Summary
- 변경 사항 요약 (bullet points)

## Test plan
- [ ] 확인 항목

🤖 Generated by [Agent Name]
```

## 승인 프로세스

1. AI Agent가 PR 생성
2. JK가 리뷰 및 피드백
3. 필요시 AI Agent가 수정
4. JK가 승인 및 머지
