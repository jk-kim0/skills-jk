# Agent PR Creation Workflow

Use this reference when adding or maintaining a GitHub Actions workflow that lets AI agents create pull requests in `querypie/outbound-agent`.

## Canonical pattern

The repository can provide a manual `workflow_dispatch` workflow at `.github/workflows/create-pr.yml` for agent-created PRs.
Use `skills-jk`'s `.github/workflows/create-pr.yml` as the baseline pattern, then adapt it to Outbound Agent conventions.

Recommended inputs:

- `branch`: source branch to open the PR from.
- `title`: PR title.
- `body`: Markdown PR body.
- `base`: base branch, default `main`.
- `draft`: whether to create a draft PR.

Recommended safeguards:

1. Checkout the source branch with full enough history for a base comparison.
2. Reject `branch == base`.
3. Fetch the base branch and reject no-diff PR creation.
4. Query open PRs for the same `head`/`base`; if one exists, print its URL and exit successfully instead of creating a duplicate.
5. Write the PR body to a temporary file and call `gh pr create --body-file`, avoiding shell quoting issues with markdown/backticks.
6. Keep workflow permissions narrow: `contents: read`, `pull-requests: write` unless the workflow also commits or pushes.
7. Run `actionlint .github/workflows/create-pr.yml` and `git diff --check` before committing.

## YAML pitfall

Avoid embedding a bare `---` line at the start of a heredoc inside a `run: |` block if the local YAML parser/linter treats it as a second YAML document.
Prefer appending the markdown separator with `printf`, for example:

```bash
printf '%s\n%s\n' '---' '🤖 Generated by github-actions[bot] for AI Agent PR creation.' >> "$body_file"
```

## Reporting

When creating the PR for this workflow, report:

- workflow path,
- source branch and PR URL,
- copied/adapted source reference from `skills-jk`,
- validation commands (`actionlint`, `git diff --check`),
- current CI/check state without passively waiting on long deploy checks.
