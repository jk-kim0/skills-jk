# Stage Finish

이 문서는 사용자가 단계 완료를 요청할 때의 사용자-facing 흐름을 정의한다.

사용자는 내부 검사 이름을 알 필요가 없다. 다음 표현은 모두 같은 의도로 본다.

- `마무리해줘`
- `이 단계 끝내줘`
- `다음 단계로 넘겨줘`
- `다음 단계로 넘어가도 되는지 확인해줘`

## User-Facing Flow

사용자가 단계 마무리를 요청하면 Agent는 다음 순서를 따른다.

1. `finish-stage.sh <case-id> <stage>`를 실행한다.
2. 결과가 `보완 필요`이면 사용자에게 보완할 항목을 짧게 설명한다.
3. Agent가 수정할 수 있는 문서 문제는 직접 수정한다.
4. 사용자의 결정이 필요한 항목은 질문한다.
5. 결과가 `내용 검토 필요`이면 Agent가 내용 검토 문서를 작성한다.
6. 내용상 완료 가능하면 내부 완료 처리 스크립트를 실행한다.
7. 사용자에게는 `마무리 완료`, `보완 필요`, `진행 승인 필요` 중 하나로 말한다.

## User Language

사용자에게 내부 검사 이름, 내부 상태값, 내부 스크립트 이름을 먼저 말하지 않는다.

사용자가 내부 동작을 물어본 경우에만 세부 용어를 설명한다.

## Internal Procedure

`finish-stage.sh`는 사용자-facing 진입점이다.

내부적으로는 기존 단계 완료 점검을 실행하고, 문제가 없으면 내용 검토 초안 파일을
준비한다.

내용 검토 파일은 `.agents/runs/sdlc-stage/<case-id>/<stage>-content-review.md`에 둔다.

Agent는 내용 검토를 시작하기 전에 현재 단계 skill의
`assets/prompts/stage-content-review.md`를 읽는다.

Agent는 내용 검토 초안 파일을 작성한 뒤 내부 완료 처리 스크립트를 실행한다.

스크립트가 자동으로 LLM 판단을 수행하지는 않는다. 의미 판단은 Agent가 문서를 읽고
수행한다.
