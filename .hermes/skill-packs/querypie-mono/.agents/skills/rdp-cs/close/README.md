# RDP CS 종료 안내 템플릿

> **이 문서의 역할**: CS 대응 시 Claude가 참조하는 **종료 안내 템플릿**.

## 템플릿 목록

| 유형 | 템플릿 | 사용 시점 | 키워드 |
|:----:|--------|----------|--------|
| bug | 버그 확인 및 패치 제공 | 버그 확인 후 핫픽스 제공 | bug, 버그, patch |
| env | 환경 문제 해결 안내 | 환경 문제 해결 후 | env, 환경, resolved |
| schedule | 버그 수정 일정 안내 | 수정 버전 대기 필요 | schedule, 일정, version |
| more-data | 추가 자료 요청 | 분석에 추가 정보 필요 | more, 추가, data |
| infra | 고객사 인프라 문제 | 고객사 측 조치 필요 | infra, 인프라, customer |
| normal | 정상 동작 확인 | 문제 없음 확인 | normal, 정상, ok |

## 버그 vs 환경 문제 판단 기준

### 버그로 판단하는 경우

| 판단 근거 | 설명 |
|-----------|------|
| 동일 환경에서 재현 | 내부 테스트 환경에서 동일 OS로 재현됨 |
| 특정 버전에서만 발생 | 이전 버전에서는 정상 |
| 코드 수정으로 해결 | Fallback 로직, 버그 수정 패치로 해결 |
| 다수 고객사 동일 증상 | 환경이 다른 여러 고객사에서 동일 패턴 |

**대표 버그**:
- Windows Schannel ALPN 버그 (QCP-3828) -> 핫픽스
- mouse->time overflow (QCP-4704) -> 버전업

### 환경 문제로 판단하는 경우

| 판단 근거 | 설명 |
|-----------|------|
| 고객사 환경 특수성 | 특정 네트워크/보안 정책에서만 발생 |
| 설정 변경으로 해결 | DNS, 방화벽, 레지스트리 수정으로 해결 |
| 다른 고객사는 정상 | 동일 버전인데 다른 곳은 정상 |
| OS/인프라 제약 | 구형 OS 미지원, 인증서 만료 등 |

**대표 환경 문제**:
- DNS 미등록 (QCP-3857) -> hosts/DNS 설정
- Custom Listener 누수 (QCP-4336) -> 클린 설치
- TLS 1.2 미지원 (QCP-4556) -> TLS 활성화

## 템플릿: 버그 확인 및 패치 제공 (bug)

```
@@{파트너 담당자} cc. @@{관련자}

안녕하세요, 전달 주신 로그 분석하였으며, {증상 설명}을 확인하였습니다.

원인은 {근본 원인}이며, 현재 내부에서 동일 환경으로 재현 완료한 상태입니다.

해당 이슈를 해결한 Server Agent 핫픽스({버전})를 공유드립니다.

[수정 내용]
* {수정 사항 1}
* {수정 사항 2}

적용 후 정상 동작 여부 확인 부탁드립니다.
감사합니다.
```

## 케이스별 고객 안내 예시

### 케이스 A: TLS/ALPN (버그)

```
원인은 Windows Schannel 버그이며, 동일 OS 버전으로 재현 완료했습니다.

Fallback 로직을 구현한 Server Agent 핫픽스를 공유드립니다.
* QueryPie Server에서 일부 Windows Server 2019에 TLS Handshake 실패하지 않도록 개선
```

### 케이스 B: DNS resolve (환경)

```
해당 건 조치 완료되었습니다.

- 원인: 쿼리파이 도메인이 Route53에 미등록
- 조치: Windows Server hosts 파일에 도메인-ALB 매핑 추가
```

### 케이스 C: Listener 누수 (환경)

```
RDP Services 재기동 절차 누락으로 Custom Listener Port 누수가 발생했습니다.

RDP-QP-1 등의 Custom Listener는 Agent 삭제 후 RDP Services 재기동이 필요합니다.
services.msc에서 Remote Desktop Service를 재시작하면 포트 할당이 초기화됩니다.
```

### 케이스 E: mouse->time overflow (버그)

```
분석 결과 전달 드립니다.

1. 본 이슈는 mouse click 발생 시각 계산 로직에 버그가 있어서,
   부팅 후 50일 이상 된 서버에서 발생합니다.

   - 관리자페이지상 10분인데 5분만에 세션 종료되는 현상

2. 서버에이전트 버전업(11.5.1+)이 필요합니다.
```
