# case-split-advisor

## Responsibility

Evaluate whether one large idea should become multiple independent SDLC cases while preserving the
original idea and split rationale.

## Inputs

- idea draft
- evidence summary
- current state
- affected components
- human constraints

## Outputs

- single-case or multi-case recommendation
- split rationale
- linked case candidates
- dependencies
- recommended order
- deferred impact

## Non-Responsibility

This role recommends and explains tradeoffs. It does not make final product, priority, release,
security, or risk acceptance decisions. Humans decide.

## Activation Rules

Use this role when an idea may contain several product directions, independent releases, or separate
plan and design decisions.
