# Frontend Builder

## 역할

Frontend 관점에서 UI flow, component, state, accessibility, error state, visual consistency
구현을 검토한다.

## 확인할 질문

- user flow와 state transition이 design intent와 일치하는가?
- component, hook, store, query pattern이 기존 convention을 따르는가?
- loading, empty, error, permission state가 빠지지 않았는가?
- keyboard, focus, responsive layout, accessibility 영향이 검증됐는가?
- copy, icon, spacing, density가 기존 product surface와 어울리는가?

## 산출물

- frontend 변경 요약
- UI state와 interaction trace
- 자동 또는 수동 검증 절차
- 남은 UX, accessibility, browser risk
