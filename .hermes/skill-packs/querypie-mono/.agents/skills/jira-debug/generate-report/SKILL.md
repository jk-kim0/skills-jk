# Generate Report Skill

모든 분석 결과를 종합하여 구조화된 마크다운 리포트를 생성합니다.

## 사용법

```
/jira-debug-generate-report
```

이전 스킬들의 분석 결과가 컨텍스트에 있어야 합니다:
- `/jira-debug-fetch-ticket` - 티켓 정보
- `/jira-debug-identify-components` - 컴포넌트 식별
- `/jira-debug-analyze-issue-detail` - 에러 상세 분석
- `/jira-debug-search-code` - 코드 검색 결과

## 리포트 섹션 설명

### 필수 섹션

1. **Ticket 정보**: 기본 티켓 메타데이터
2. **에러 분석**: 에러 타입, 메시지, HTTP 상태
3. **관련 컴포넌트**: 식별된 코드 컴포넌트 목록
4. **원인 조사 결과**: 코드 검색에서 발견된 잠재적 원인

### 선택 섹션 (정보가 있는 경우만)

- **재현 단계**: 티켓에 재현 단계가 있는 경우
- **호출 체인**: 스택 트레이스가 있는 경우
- **추가 조사 필요**: 불확실한 부분이 있는 경우

## 출력

마크다운 형식의 리포트 텍스트를 반환합니다.
이 리포트는 `/send-slack` 스킬에서 Slack 메시지로 전송됩니다.

## 리포트 품질 기준

### 포함해야 할 것
- 구체적인 파일 경로와 라인 번호
- 실제 코드 스니펫
- 명확한 권장 조치

### 피해야 할 것
- 추측성 내용 (확실하지 않으면 "추가 조사 필요"로 표시)
- 너무 긴 코드 블록 (핵심 부분만)
- 중복된 정보
