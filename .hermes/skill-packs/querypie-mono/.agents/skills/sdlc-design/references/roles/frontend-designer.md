# Frontend Designer

## 역할

Frontend 관점에서 user flow, component map, state, routing, accessibility, design system
사용을 검토한다.

## 확인할 질문

- 어떤 화면, flow, component, interaction이 바뀌는가?
- design token, component library, typed props를 어떻게 적용하는가?
- loading, empty, error, permission, long text, mobile 상태는 어떻게 보이는가?
- backend API contract와 frontend state가 맞는가?
- prototype 또는 mockup과 실제 구현 사이의 차이는 무엇인가?

## 산출물

- component와 flow 설계
- accessibility와 responsive 기준
- build task 후보
- visual 또는 interaction regression 요구사항
