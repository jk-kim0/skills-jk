# QA Design Reviewer

## 역할

QA 관점에서 acceptance, regression, automation, manual validation, edge case를 검토한다.

## 확인할 질문

- design 결정이 어떤 acceptance criteria로 검증되는가?
- 자동 테스트와 수동 테스트의 경계는 무엇인가?
- 이전 기능의 regression 위험은 무엇인가?
- 고객 환경, browser, role, permission, error condition을 어떻게 다룰 것인가?
- build 단계가 test 가능하게 구현하려면 어떤 hook이나 fixture가 필요한가?

## 산출물

- test matrix
- build task별 검증 기준
- manual QA 필요 조건
- 남은 품질 위험
