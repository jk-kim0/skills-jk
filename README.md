# skills-jk

`skills-jk`는 사람이 읽고, AI Agent도 함께 따를 수 있도록 만든 스킬·태스크·프로젝트 관리 저장소입니다.
GitHub Actions와 self-hosted runner를 통해 자동화 워크플로우를 실행합니다.

## 이 저장소의 목적

- 재사용 가능한 Skill을 정의합니다.
- 구체적인 태스크를 관리합니다.
- 장기 프로젝트의 진행과 산출물을 추적합니다.

## 공통 가이드

사람과 AI Agent 모두 다음 원칙을 함께 따릅니다.

- 변경은 작고 명확하게 유지합니다.
- 실제 동작의 기준은 코드와 스킬 문서입니다.
- 정책과 절차는 가능한 한 한 곳에만 두고, 나머지는 요약하거나 연결합니다.
- 문서가 구현과 어긋나면, 먼저 기준 문서를 맞춥니다.

## 사람을 위한 안내

이 저장소를 처음 볼 때는 아래 순서로 읽으면 됩니다.

1. `README.md` — 저장소 소개와 전체 구조
2. `AGENTS.md` — AI Agent 작업 규칙과 운영 가이드
3. `skills/<name>/SKILL.md` — 개별 작업용 절차

주요 디렉터리:

- `skills/` — 재사용 가능한 Skill
- `tasks/` — 개별 작업 기록
- `projects/` — 장기 프로젝트 문서
- `docs/` — 설명, 정책, 운영 문서
- `reports/` — 결과 요약과 산출물

## AI Agent 설정 가이드

AI Agent가 이 저장소를 작업할 때는 다음을 따르면 됩니다.

- 먼저 `AGENTS.md`를 읽습니다.
- 필요한 경우 `skills/<name>/SKILL.md`를 확인합니다.
- 저장소 규칙에 맞는 최소 변경만 수행합니다.
- 작업 후에는 관련 검증을 수행합니다.
- 브랜치 기반 작업은 새 worktree를 우선 사용합니다.

## 문서 운영 원칙

- README.md는 사람을 위한 소개와 시작점입니다.
- AGENTS.md는 Codex, Claude Code 같은 AI Agent의 작업 가이드입니다.
- 둘 다 공통 원칙을 공유하되, 중복되는 긴 절차는 두지 않습니다.
- 새 규칙은 우선 AGENTS.md에 두고, 필요한 경우 README.md에서 짧게 안내합니다.
