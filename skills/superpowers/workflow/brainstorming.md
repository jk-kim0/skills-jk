# Brainstorming

## 목적

구현 전 요구사항과 설계를 구체화해 재작업을 줄입니다.

## 수행 절차

1. 목표/성공 기준 확인
2. 제약(성능, 보안, 일정, 호환성) 정리
3. 대안 2~3개 비교
4. 선택안과 제외 사유 확정
5. 산출물을 계획 입력으로 전달

## 출력 형식

- 문제 정의
- 성공 기준
- 제약 조건
- 대안 비교
- 선택안
- 미해결 질문
